# MyVCS — Java Version Control System

MyVCS — это локальная децентрализованная система контроля версий, вдохновленная внутренней архитектурой Git.

## Реализованные возможности

Система поддерживает полный базовый цикл работы с версиями кода:

* **`init`** — инициализация пустого репозитория (создание скрытой директории `.myvcs`).
* **`status`** — отслеживание состояния рабочей директории: новые, измененные (staged/unstaged) и удаленные файлы.
* **`add`** — добавление файлов в Staging Area (Индекс) с вычислением их хешей.
* **`commit`** — фиксация состояния с сохранением автора, даты и сообщения. Поддержка коммитов слияния (с двумя родителями).
* **`log`** — просмотр истории коммитов с обходом графа предков.
* **`checkout`** — "машина времени": восстановление файлов рабочей директории до состояния любого прошлого коммита или переключение между ветками.
* **`branch`** — легковесное ветвление: создание, удаление и просмотр списка веток.
* **`merge`** — трехстороннее слияние (3-way merge) с автоматическим поиском общего предка (LCA). Включает обнаружение конфликтов и расстановку маркеров `<<<<<<<` / `>>>>>>>` для ручного разрешения.

## Архитектура приложения

Проект построен на базе классической **многослойной архитектуры (Layered Architecture)** с жестким разделением зон ответственности. Направление зависимостей строго сверху вниз: CLI -> Core -> Storage.

### 1. Слой представления (CLI Layer)
Пакет `ru.hse.cli`. Отвечает исключительно за взаимодействие с пользователем.
* Реализован паттерн **Command**. Каждая команда реализует единый интерфейс.
* `CommandParser` выполняет маршрутизацию пользовательского ввода и передает управление в слой бизнес-логики.

### 2. Слой бизнес-логики (Core Layer)
Пакет `ru.hse.core`. Сердце системы.
* **`VcsService`** — фасад, оркестрирующий высокоуровневые операции (создание коммитов, слияние, статус).
* **`IndexManager`** — управляет Staging Area. Хранит плоскую отсортированную мапу путей и хешей файлов, готовых к коммиту.
* **`ReferenceManager`** — управляет легковесными указателями (ветками) и файлами `HEAD` / `MERGE_HEAD`.
* **`MergeResolver`** — алгоритмическое ядро для обхода графа коммитов в ширину (BFS) с целью поиска общего предка при слиянии веток.

### 3. Слой хранения (Storage Layer)
Пакет `ru.hse.storage`. Реализует паттерн **Content-Addressable Storage** (хранилище, адресуемое по содержимому).
* Все файлы сжимаются алгоритмом **Zlib** (`DeflaterOutputStream`) перед записью на диск для экономии места.
* Именем файла на диске выступает **SHA-256 хеш** от его содержимого (с префиксом типа объекта).
* Первые два символа хеша становятся именем директории (например, `.myvcs/objects/a1/b2c3d4...`), чтобы избежать лимитов ОС на количество файлов в одной папке.

## Доменная модель

В пакете `ru.hse.model` описаны иммутабельные сущности (используются Java `record`), которые хранятся в базе данных объектов:

1. **`Blob` (Binary Large Object)** — хранит исключительно сырое содержимое файла. Ничего не знает о своем имени или пути.
2. **`Tree`** — представляет директорию. Хранит список ссылок (хешей) на вложенные `Blob` и `Tree`, а также их реальные имена и права доступа.
3. **`Commit`** — снимок состояния всего проекта. Хранит ссылку на корневое `Tree`, метаданные (автор, время, сообщение) и список хешей родительских коммитов.

## Пример использования

```bash
# Инициализация
java -jar myvcs.jar init

# Создание файла и добавление в индекс
echo "Hello World" > Main.java
java -jar myvcs.jar add Main.java
java -jar myvcs.jar status

# Первый коммит
java -jar myvcs.jar commit -m "Initial commit"

# Создание новой ветки и переключение на нее
java -jar myvcs.jar branch feature-auth
java -jar myvcs.jar checkout feature-auth

# ... работа с кодом, добавление коммитов ...

# Возврат в master и слияние фичи
java -jar myvcs.jar checkout master
java -jar myvcs.jar merge feature-auth
```

## 🛠 Запуск и сборка

Требования: **Java 14+** (из-за использования `record` и современных возможностей `switch`).

Для компиляции проекта:
```bash
javac -d out $(find src -name "*.java")
```

Для запуска:
```bash
java -cp out ru.vcs.Main <команда>
```
